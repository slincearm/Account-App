優化圓餅圖顯示與金額格式

## 修改內容

### 圓餅圖視覺優化
- **標籤換行顯示**：使用 SVG `<tspan>` 實現類別名稱與百分比分行顯示
- **尺寸調整**：縮小圓餅圖尺寸（innerRadius: 50→35, outerRadius: 70→50），留出更多空間給外圍標籤
- **顏色一致性**：外圍標籤顏色與類別顏色一致，提升視覺辨識度
- **中心文字優化**：調整中心統計數字和說明文字的大小與位置

### 金額格式優化
- **統計數據移除小數點**：總支出、類別金額、日期總計、已支付、結餘等統計數據改為整數顯示（`.toFixed(0)`）
- **帳目保留一位小數**：實際費用金額與每人分攤金額保留一位小數（`.toFixed(1)`），確保帳目精確度
- **顯示邏輯優化**：區分統計數據（簡潔顯示）與實際帳目（精確顯示）

### 技術實現
- **自定義標籤渲染函數**：使用 `label={(props) => {...}}` 取代簡單字串
- **SVG 元素組合**：使用 `<text>` 和 `<tspan>` 實現真正的多行顯示
- **動態定位計算**：根據角度和半徑動態計算標籤位置
- **類型安全**：添加 undefined 檢查（`|| 0`）避免運行時錯誤
- **顏色提取**：從 `payload.categoryKey` 獲取類別並套用對應顏色

### 響應式改進
- 縮小圓餅圖在 440px 寬度手機螢幕上能完整顯示所有標籤
- 標籤文字不會被截斷或超出容器範圍

### 修改檔案
- `src/pages/GroupDetail.tsx` - 圓餅圖組件與金額格式

### 使用者體驗提升
- 標籤清晰易讀，不再因文字過長被截斷
- 統計數據簡潔（整數），帳目精確（一位小數）
- 每個類別標籤顏色與圓餅圖顏色一致，視覺關聯性更強
- 手機螢幕上顯示效果更佳

### 安全性考量
- 管理者身份在後端驗證（admin collection）
- 防止管理者誤修改自己的權限
- 批次操作使用 Promise.all 確保原子性

## Firebase 設定要求

需在 Firestore 建立 admin collection：
{
  "adminId": {
    "email": "admin@example.com"
  }
}
